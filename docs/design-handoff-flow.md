# Design Handoff Flow

このドキュメントは、Figma MCP 前提ではないデザイン支給案件で、画像ベースで実装を進めるための運用フローです。

## 前提

- デザインは Figma データではなく画像支給になることがある
- Figma MCP を前提にした厳密な座標抽出はできない
- そのため、画像と補足情報をもとに段階的に再現度を上げていく

## 基本方針

- 最初に全体画像を見てページ構造を作る
- その段階では骨組みと大まかな見た目まで進める
- 細部はセクションごとの画像支給後に詰める
- 画像素材は最後に差し込み、最初はプレースホルダで進める

## 推奨フロー

### Step 1. 全体画像を確認する

最初に PC / SP の全体画像を確認する。

この段階で決めるもの:

- ページ全体のセクション構成
- 見出し階層
- コンテナ幅
- 背景色の切り替え
- 主要なレイアウトパターン
- ベースカラー 6 色
- フォントの方向性

この段階では、まだ細かい余白や文字サイズを確定しない。

### Step 2. 骨組みを実装する

全体画像をもとに、まずページの骨組みを作る。

ここで進めてよいもの:

- semantic HTML
- section 分割
- 見出し構造
- 大枠のレイアウト
- `l-container` などのレイアウト設定
- おおまかなタイポグラフィ
- 背景の切り替え
- PC / SP の大きな並びの違い

この段階は、完成品ではなく「低解像度のデザイン実装」を作るイメージで進める。

### Step 3. 画像はプレースホルダで置く

画像素材は最後に差し替える前提で、先にプレースホルダを置く。

ルール:

- 単なる空箱ではなく、最終画像に近い比率で置く
- `aspect-ratio` か固定の width / height 前提を決める
- 必要なら `object-fit` の方針も先に決める
- FV、商品画像、バナー、アイコンでプレースホルダの扱いを分ける

プレースホルダ運用の目的:

- レイアウトを先に固める
- 素材待ちで実装を止めない
- 後から差し替えても崩れにくい状態を作る

### Step 4. セクションごとの画像で精度を上げる

全体の骨組みができた後に、セクション単位で詳細画像を確認する。

ここで詰めるもの:

- 余白
- 文字サイズ
- 行間
- ボタンのサイズ感
- 細かい装飾
- absolute 配置の微調整
- PC / SP での崩し方

### Step 5. 本画像へ差し替える

最後にプレースホルダを本画像へ差し替える。

差し替え時に確認するもの:

- `alt`
- lazy load の要否
- LCP 対象画像かどうか
- 画像の比率崩れ
- CLS の有無

## 再現性を上げるために必要な補足情報

画像だけだとブレやすいので、可能なら次の情報をテキストで渡す。

- 使用フォント
- ベースカラー 6 色
- コンテンツ最大幅
- セクション間余白の基準
- ボタンの hover 方針
- アニメーション有無
- SP 時のルール

## 推奨する支給順

1. PC 全体画像
2. SP 全体画像
3. セクションごとの拡大画像
4. 使用画像素材
5. 追加の補足テキスト

## このフローでの再現優先順位

優先順位は次の順で考える。

1. レイアウト構造
2. 余白とサイズ感
3. 配色
4. タイポグラフィ
5. 装飾
6. アニメーション

## 画像支給時の注意

- 画像だけで全てのサイズを断定しない
- まず全体構造を優先する
- 1 セクションだけ完璧に詰めすぎず、まず全体バランスを合わせる
- 画像差し替え後に必ず最終調整を行う
